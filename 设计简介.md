# 2.1 模块一：校园概览

## 2.1.1 模块定位

用于展示校园的**总体资源、空间分布、人员概况、运行态势**。
这是默认首页，重点是“看全局、看空间、看基础运行”。

## 2.1.2 中间场景设计

### 场景核心效果

使用 **POI点位 API** 展示校园重点建筑和功能节点。

### 建议展示对象

* 教学楼
* 宿舍楼
* 食堂
* 图书馆
* 体育馆/操场
* 实验楼
* 行政楼
* 校门
* 停车场
* 医务室
* 超市/生活服务点

### POI效果设计

每个建筑 POI 建议包含以下视觉效果：

1. **图标区分**

   * 教学楼：书本/教学楼图标
   * 宿舍：宿舍床位图标
   * 食堂：餐盘图标
   * 图书馆：书架图标
   * 体育馆：运动图标
   * 行政楼：办公楼图标

2. **呼吸光圈**

   * 静态状态：低频呼吸
   * 选中状态：双层扩散光圈
   * 告警状态：闪烁 + 外环脉冲

3. **竖向锚点线**

   * 从地面拉起到图标位置
   * 强化三维空间定位感

4. **标签牌**

   * 默认显示建筑名称
   * 近景显示建筑名称 + 类型
   * 选中后显示关键指标，如人数、使用率、设备数

5. **建筑高亮**

   * 点击 POI 后，建筑模型描边高亮
   * 镜头 FlyTo 到该建筑
   * 建筑顶部可附加轻微泛光

6. **聚合显示**

   * 远景下同类 POI 聚合
   * 近景下自动拆分

# 2.2 模块二：智能检测

## 2.2.1 模块定位

## 2.2.2 中间场景设计

### 场景核心效果

使用 **区域热力图 API** 展示检测结果。

### 可检测主题建议

* 人流密度
* 重点区域聚集
* 夜间异常逗留
* 烟雾/火点异常
* 未佩戴安全装备
* 校园垃圾暴露/环境问题
* 车辆乱停放
* 黑名单人员预警
* 实验室异常温湿度

### 热力图效果设计

1. **分区热度渲染**

   * 绿色：正常
   * 黄色：轻度异常
   * 橙色：中度异常
   * 红色：高风险

2. **网格热力 / 面域热力**

   * 操场、食堂、宿舍区适合面域热力
   * 走廊、主干道适合网格热力

3. **动态跳变**

   * 热点值变化时平滑过渡
   * 高风险区域伴随脉冲光效

4. **时间轴联动**

   * 可回放某时间段检测结果变化

5. **区域点击**

   * 点击热区后右侧显示该区域检测详情
   * 支持钻取到摄像头列表或事件列表

# 六、落地时的表现建议

为了让这个页面更像“智慧校园大屏”，而不是普通 BI 页，建议把以下几件事做出来：

1. **中心场景永远是主角**
   左右面板只做辅助说明，不要压过中间三维场景。

2. **模块切换必须有明显图层变化**
   不能只是换几张图表，否则用户感觉不到四个模块的差异。

3. **每个模块至少有一个专属场景效果**

   * 校园概览：POI
   * 智能检测：热力图
   * 设备运维：巡检路线 + 调度迁徙线
   * 安防管理：区域轮廓 + 风险闪烁

4. **图表数量控制**
   每侧 3 到 4 个面板最合适，太多会挤。

5. **强调联动**
   图表点击 -> 场景高亮 -> 详情弹窗，这个链路一定要完整。

# 七、适合直接给研发的简版结论

这套大屏建议定义为：

* **一个三维校园底座**
* **四个业务模块页签**
* **四套左右面板内容**
* **四类核心场景能力**

  * POI点位
  * 区域热力图
  * 巡检/调度路线
  * 区域轮廓分区
* **一套统一联动机制**

  * 图表联动场景
  * 场景联动详情
  * 告警联动弹窗

这样做出来，既有你图里的“大屏感”，又有真正可交付的业务逻辑。
